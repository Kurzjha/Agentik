# Frontend Rules

- Prefer deliberate UI structure over generic boilerplate layouts.
- Use responsive layouts that work on desktop and mobile.
- Avoid default system font stacks unless the repository already uses them.
- Use clear visual hierarchy, intentional spacing, and consistent component patterns.
- Add accessibility labels, keyboard-friendly interactions, and semantic structure.
- When a task affects visible UI, validate the rendered result, not only the code.
- Keep state and data flow simple; avoid unnecessary abstraction in small interfaces.
