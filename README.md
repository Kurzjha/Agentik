# Agentik

Agentik — это локальный CLI-harness для работы LLM с проектом на файловой системе. Он хранит историю сессий, собирает системный промпт из локального контекста, ограничивает область изменений заданной директорией проекта и умеет работать как в интерактивном режиме, так и в режиме одного запроса.

По умолчанию проект настроен на `GigaChat`, но код также поддерживает провайдеров с токеном через `MODEL_TOKEN`.

## Возможности

- запуск в интерактивном и одноразовом CLI-режиме
- хранение истории сессий в JSONL
- ограничение рабочей области через `--project-root`
- профили контекста для разных типов задач
- планирование и подмешивание исследовательского контекста
- инструменты для чтения, записи, создания директорий и запуска shell-команд

## Структура проекта

- `main.py` — точка входа CLI
- `agent.py` — основной цикл агента и оркестрация
- `config.py` — загрузка настроек, окружения и сборка системного промпта
- `tools.py` — локальные инструменты агента
- `memory.py` — память сессий и аналитика по истории
- `planning.py` — кэш и вспомогательная логика планирования
- `research.py` — сбор исследовательского контекста
- `context_profiles.py` — определение профилей контекста
- `profiles/` — шаблоны и правила профилей
- `Generated_Project/` — рабочая директория по умолчанию, куда агент пишет результаты
- `.claude/projects/` — сохраненные состояния сессий

## Требования

- Python 3.10 или новее
- доступ к совместимому model backend
- зависимость `python-dotenv` из `requirements.txt`

## Установка

1. Создайте и активируйте виртуальное окружение.
2. Установите зависимости:

```bash
pip install -r requirements.txt
```

3. Создайте файл `.env` в корне репозитория.

Для стандартной конфигурации `GigaChat` из `settings.json`:

```env
Authorization_key=your_key_here
```

Если используется провайдер с токеном:

```env
MODEL_TOKEN=your_token_here
```

## Конфигурация

Основные параметры runtime лежат в `settings.json`.

Ключевые поля:

- `base_url` — URL API модели
- `model` — имя модели
- `project_root` — директория, доступная агенту для записи
- `session_root` — каталог хранения истории сессий
- `library_root` — каталог вспомогательных материалов
- `permission_mode` — политика выполнения shell-команд: `plan`, `auto` или `bypass`

Значения по умолчанию:

- `project_root`: `Generated_Project`
- `session_root`: `.claude/projects`

## Использование

Интерактивный режим:

```bash
python main.py
```

Один запрос из CLI:

```bash
python main.py "Создай Flask-приложение с формой обратной связи"
```

С указанием сессии и рабочей директории:

```bash
python main.py "Добавь тесты" --session demo --project-root Generated_Project
```

Headless-режим с чтением запроса из файла:

```bash
python main.py --headless-file prompt.txt
```

## Параметры CLI

- `prompt` — текст запроса пользователя
- `--session` — идентификатор сессии для JSONL-истории
- `--permission-mode` — `plan`, `auto` или `bypass`
- `--project-root` — директория, которую агент может изменять
- `--headless-file` — файл, из которого читается запрос

## Команды интерактивного режима

- `/clear` — очистить историю текущей сессии
- `/rewind <message_index>` — откатить историю к указанной позиции
- `/exit` или `/quit` — выйти из интерактивного режима

## Тесты

В репозитории есть, в частности, такие тесты:

- `test_agent.py`
- `test_context_profiles.py`
- `test_planning_memory.py`

Запуск:

```bash
python -m unittest
```

## Примечания

- Агент должен читать и изменять файлы только внутри `PROJECT_ROOT`.
- Выполнение shell-команд зависит от `permission_mode`.
